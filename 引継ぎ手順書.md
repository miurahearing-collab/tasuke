# 「タスケ」アプリ 新PC引き継ぎ手順書

**作成日：2026年4月18日**

---

## はじめに読んでください

### このアプリについて

「タスケ」は、チームのタスク・スケジュールを管理するWebアプリです。
Firebaseというクラウドサービスにデータが保存されているため、**PCを変えてもデータは消えません。**
新しいPCでは「開発環境を整えてコードを動かす」という作業だけが必要です。

### 引き継ぎの全体の流れ

```
【今のPC】
  ↓ 最新コードがGitHubに保存済み ✅（完了）

【新しいPC】
  STEP 1：必要なソフトを4つインストールする
  STEP 2：GitHubからコードをダウンロードする
  STEP 3：アプリを動かすための準備をする
  STEP 4：アプリを起動して確認する
  STEP 5：Claude Code（AI）を使えるようにする
```

> **ポイント：** データ（タスク・メンバーなど）はクラウドに保存済みなので移行不要です。
> 「コードを動かす環境」だけを新PCに作ります。

# STEP 1：新しいPCに必要なソフトをインストールする

必要なソフトは以下の4つです。**必ず順番通りにインストールしてください。**

| # | ソフト名 | 役割 |
|---|----------|------|
| 1 | Git | コードのバージョン管理・GitHubからダウンロード |
| 2 | Node.js | JavaScriptを動かすエンジン |
| 3 | Visual Studio Code | コードを書くためのエディタ |

## 1-1. Git のインストール

Gitは「コードの変更履歴を管理するツール」です。GitHubからコードをダウンロードするためにも必要です。

### インストール手順

**① ダウンロード**

ブラウザで以下のURLを開く：
```
https://git-scm.com/download/win
```

ページが開いたら「**64-bit Git for Windows Setup**」というリンクをクリック。
自動的にダウンロードが始まります。

**② インストール**

ダウンロードした `.exe` ファイルを**ダブルクリック**して起動。

インストール中は以下の画面がいくつか出てきますが、**すべて「Next」ボタンをクリックするだけ**でOKです。
（設定を変える必要はありません）

最後に「**Install**」ボタンが出たらクリック。完了したら「**Finish**」をクリック。

**③ インストール確認**

Windowsの「スタートメニュー」を右クリック → 「**ターミナル**」または「**コマンドプロンプト**」を開く。

以下の文字を入力して **Enterキー** を押す：
```
git --version
```

以下のように表示されればインストール成功です：
```
git version 2.47.0.windows.1
（バージョン番号は多少違っても問題なし）
```

> **うまくいかない場合：** PCを再起動してからもう一度試してください。

## 1-2. Node.js のインストール

Node.jsは「JavaScriptプログラムを動かすエンジン」です。アプリを開発・起動するために必要です。

### インストール手順

**① ダウンロード**

ブラウザで以下のURLを開く：
```
https://nodejs.org/ja/
```

ページの中央に大きなボタンが2つあります。**左側の「LTS推奨版」** をクリックしてダウンロード。

> **「LTS」とは？** Long Term Support（長期サポート）版のことで、安定して使えるバージョンです。

**② インストール**

ダウンロードした `.msi` ファイルを**ダブルクリック**して起動。

こちらも**すべてデフォルトのまま「Next」をクリック**し続けるだけ。
「**Install**」が出たらクリック。完了したら「**Finish**」をクリック。

**③ インストール確認**

ターミナル（またはコマンドプロンプト）を**一度閉じて、新しく開き直す**。

以下を入力して **Enter**：
```
node --version
```
以下のように表示されればOK：
```
v20.19.0
（v18以上であれば問題なし）
```

続けて以下も確認：
```
npm --version
```
```
10.9.0
（何かバージョン番号が表示されればOK）
```

## 1-3. Visual Studio Code のインストール

Visual Studio Code（略：VSCode）は「コードを書くためのメモ帳の高機能版」です。

### インストール手順

**① ダウンロード**

ブラウザで以下のURLを開く：
```
https://code.visualstudio.com/
```

青い「**Download for Windows**」ボタンをクリック。

**② インストール**

ダウンロードした `.exe` ファイルをダブルクリックして起動。

インストール中は以下だけ設定を変更してください：

- 「追加タスクの選択」という画面が出たら、以下の2つにチェックを入れる（他はデフォルトのまま）：
  - ✅「**エクスプローラーのファイルコンテキストメニューに[Codeで開く]アクションを追加する**」
  - ✅「**エクスプローラーのディレクトリコンテキストメニューに[Codeで開く]アクションを追加する**」

あとは「次へ」→「インストール」→「完了」でOK。

# STEP 2：GitHubからコードをダウンロードする

このプロジェクトのコードはGitHubに保存済みです。新しいPCにダウンロードします。

## 2-1. 保存先フォルダを作る

ターミナルを開いて、以下を**1行ずつ**入力して Enter を押してください。

```
cd C:\Users\（あなたのユーザー名）
```

> **「あなたのユーザー名」部分の調べ方：**
> ターミナルを開いたときに `C:\Users\yamada>` のように表示されている「yamada」の部分がユーザー名です。

```
mkdir "AI作成アプリ"
```
```
cd "AI作成アプリ"
```

> これで `C:\Users\（ユーザー名）\AI作成アプリ` というフォルダができました。

## 2-2. GitHubからコードをダウンロードする

以下を入力して Enter：

```
git clone https://github.com/miurahearing-collab/tasuke.git
```

以下のように表示されながらダウンロードが進みます：
```
Cloning into 'tasuke'...
remote: Enumerating objects: ...
remote: Counting objects: ...
Receiving objects: 100% ...
done.
```

「**done.**」と表示されたら完了です。

`AI作成アプリ` フォルダの中に `tasuke` というフォルダができています。

> **GitHubへのログインを求められた場合：**
> GitHubのアカウント（`miurahearing-collab`）でログインしてください。

# よくあるエラーと対処法

## エラー①：`npm install` 中に赤いエラーが出る

**原因：** Node.jsのバージョンが古い  
**対処：** Node.jsのバージョンを確認し、v18未満なら最新版を再インストール
```
node --version
```

## エラー②：`npm run dev` でポートエラーが出る

**原因：** 5173番ポートが他のアプリに使われている  
**対処：** 別のポート番号で起動
```
npx vite --port 3000
```
その後、ブラウザで `http://localhost:3000` を開く

## エラー③：ログインできない

**原因①：** パスワードが違う  
→ Firebaseのパスワードリセット機能でリセット

**原因②：** アカウントが登録されていない  
→ Firebase管理画面で確認：https://console.firebase.google.com/  
プロジェクト「gen-lang-client-0592050939」→ Authentication → Users

## エラー④：データ（タスクなど）が表示されない

**確認方法：**
1. ブラウザで F12キー を押す
2. 「Console」タブを開く
3. 赤いエラーメッセージが出ていないか確認
4. エラーメッセージをClaude Codeに貼り付けて相談する

## エラー⑤：`git clone` でエラーが出る

**原因：** GitHubへのアクセス権がない  
**対処：** GitHubにログインしているか確認
```
git config --global user.name "miurahearing-collab"
git config --global user.email "（GitHubに登録したメールアドレス）"
```
